# 案例沉淀方案

## 目标

当前站点已经有“可展示的案例方向”，但还没有真实成交案例。

现阶段案例策略不是“假装做过很多项目”，而是：

- 先把案例展示结构设计好
- 先准备 3 组可落地的样板案例文案
- 用行业模板承接首批沟通
- 第一批真实客户成交后，快速替换成真实作品集

## 当前展示页对应关系

首页 `#works` 现在有 3 个案例位：

- 培训机构报名页
- 本地商家展示页
- AI 客服话术库

这 3 个方向是合理的，建议保留，原因：

- 对应当前主推服务
- 容易接到中文市场小单
- 每个方向都能做出明显的“前后差异”
- 后续能扩成更多行业案例

建议把这 3 个案例位当成第一批作品集骨架，不要频繁换方向。

## 案例卡统一结构

首页每个案例卡，建议统一按下面 6 个信息块组织：

1. 案例标题
2. 客户类型
3. 核心问题
4. 交付内容
5. 结果指标
6. 截图或链接

其中：

- 没有真实成交前，结果指标写“目标结果”
- 成交后再替换成“真实结果”

推荐文案骨架：

```text
案例标题：少儿美术暑期班报名页
客户类型：本地培训机构
核心问题：课程信息分散，家长看完后不知道怎么报名
交付内容：手机端报名页、课程亮点梳理、表单留资、微信咨询入口
结果指标：目标是让家长 30 秒内看懂课程并完成留资
截图或链接：页面首屏 / 报名区 / 咨询入口
```

## 样板案例文案

以下 3 组可以先作为展示页、私聊介绍、朋友圈图文的样板案例。

### 1. 培训机构报名页

适用行业：

- 少儿培训
- 职业培训
- 语言课程
- 体验课招募

样板案例：

```text
案例标题：暑期体验课报名页
客户类型：本地培训机构
核心问题：课程介绍发在微信群和朋友圈里，信息零散，报名路径不清楚
交付内容：活动落地页、课程卖点整理、时间地点展示、报名表单、微信咨询入口
页面重点：首屏讲清楚课程价值，中段放上课时间与适合人群，底部做表单和加微信双入口
结果指标：目标是降低家长咨询成本，让用户 1 分钟内完成报名或加微信
适合报价：199 - 499 元
```

### 2. 本地商家展示页

适用行业：

- 餐饮
- 美业
- 摄影
- 家政
- 本地服务门店

样板案例：

```text
案例标题：社区门店展示页
客户类型：本地商家
核心问题：客户刷到店铺信息后，只能看到零散图片，缺少完整服务说明和联系入口
交付内容：门店介绍页、服务项目展示、价格区块、地址导航、电话和微信二维码入口
页面重点：先说明核心服务，再展示套餐，最后给出地址、营业时间和联系方式
结果指标：目标是把“先问问看”变成“直接联系或到店”
适合报价：199 - 499 元
```

### 3. AI 客服话术库

适用行业：

- 咨询服务
- 教育招生
- 本地商家客服
- 微信私域成交

样板案例：

```text
案例标题：微信客服 FAQ 话术整理
客户类型：需要高频回复客户问题的小团队
核心问题：常见问题重复回答，报价和异议处理不统一，新人接待效率低
交付内容：FAQ 清单、标准回复模板、报价说明、异议处理话术、成交跟进模板
页面重点：把内容整理成可复制文档，也可以配一个简单展示页方便对外说明
结果指标：目标是减少重复沟通，提高首次回复速度和成交一致性
适合报价：499 - 999 元
```

## 截图替换规范

当前首页案例卡是占位图形。

第一批真实案例出现后，截图替换建议按下面规则执行。

### 一张案例至少准备 3 类图

1. 首屏图
2. 关键信息区截图
3. 留资或联系入口截图

如果是 AI 客服类案例，改为：

1. FAQ 结构图
2. 话术片段图
3. 使用场景图

### 截图内容要求

- 优先展示手机端页面，因为当前主打小单和微信流量
- 截图里要能看出行业、结构和行动入口
- 避免把客户手机号、微信号、真实姓名直接露出
- 没有授权时，对品牌名、人名、电话号码做模糊或替换

### 截图尺寸建议

- 首页案例卡横图：1200 x 900
- 微信传播长图：1242 x 2208
- 手机端局部截图：宽度 750 或 1080 即可

### 截图替换顺序

1. 先替换首页案例卡首图
2. 再补案例详情页或详情文档
3. 最后补客户评价、数据结果和项目链接

不要一开始就等“完整包装”。
先把首页占位图替换成真实截图，信任感就会明显提升。

## 首页案例文案替换规则

首页案例卡建议分两阶段更新。

### 阶段一：未成交前

使用“案例方向”口径：

- 培训机构报名页
- 本地商家展示页
- AI 客服话术库

副标题写“能解决什么问题”，不要写虚假结果。

### 阶段二：成交后

改成“行业 + 具体场景”的命名：

- 少儿美术体验课报名页
- 社区美容店项目展示页
- 留学咨询微信客服话术库

副标题改成：

- 做了什么
- 帮客户解决了什么问题

如果客户允许，可以再加一行结果：

- 3 天内收集到 27 条报名线索
- 客户咨询入口点击率明显提升
- 客服常见问题回复统一

没有客户明确授权前，不写夸张数字，不写可疑战报。

## 首批成交后的作品集沉淀流程

第一批成交后，不要只交付页面，要顺手把案例资产收集完整。

每个项目交付时，至少沉淀下面 8 项：

1. 项目名称
2. 客户行业
3. 初始需求
4. 交付范围
5. 页面截图
6. 项目链接或本地预览
7. 客户一句评价
8. 是否允许公开展示

建议在 `docs/cases/` 下为每个真实项目单独建一个文档，命名规则：

```text
docs/cases/2026-06-shaoer-meishu-signup.md
docs/cases/2026-06-community-store-page.md
```

单个案例文档建议结构：

```markdown
# 少儿美术体验课报名页

## 客户背景

## 原始问题

## 交付内容

## 页面结构

## 结果反馈

## 可公开素材

## 是否可继续复用为行业模板
```

## 首批成交后的执行动作

每完成一个真实案例，按下面顺序处理：

1. 保存最终页面截图
2. 记录交付范围和价格区间
3. 记录客户原话评价
4. 确认是否可以公开展示品牌名
5. 更新首页案例卡标题和副标题
6. 新增案例详情文档
7. 把可复用区块抽成行业模板

这一步很重要，因为你的目标不是只做一次项目，而是把每个小单变成下一单的销售素材。

## 建议的第一批案例优先级

如果现在还没有真实案例，建议优先拿下面 3 类单：

1. 培训机构报名页
2. 本地商家展示页
3. 带 FAQ 的微信客服整理

原因：

- 容易沟通
- 范围小
- 交付快
- 页面成果直观
- 最适合放进当前首页结构

## 后续落地建议

短期建议：

- 保留首页 3 个案例位不变
- 先按本文件准备样板文案
- 第一单成交后先换截图，再补详情

中期建议：

- 新增 `docs/cases/` 存真实案例
- 首页案例区从“方向展示”升级成“真实案例 + 结果”
- 单独做一个“案例详情页”或“案例长图”

长期建议：

- 按行业沉淀复用模板
- 把报价、案例、客户评价组合成标准成交材料
- 后续对外只重点卖 2 到 3 个最容易成交的行业方向
