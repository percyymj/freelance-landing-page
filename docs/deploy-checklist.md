# 部署操作清单

## 项目现状

- 当前项目是纯静态页面。
- 入口文件是 `index.html`。
- 静态资源放在 `assets/`。
- 目前不需要后端、不需要数据库、不需要构建工具。

## 本地预览

### 方式 1: 直接打开

适合快速确认页面内容。

```bash
open index.html
```

### 方式 2: 启动本地静态服务

适合检查图片路径、锚点跳转和相对路径。

```bash
python3 -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

## 部署路径选择

### 1. GitHub Pages

适合：

- 先快速上线
- 页面内容不需要后端
- 预算尽量低

优点：

- 免费
- 操作简单
- 适合静态展示页

注意：

- 默认是公开访问
- 自定义域名需要额外配置 DNS
- 国内访问稳定性一般

基本步骤：

1. 把代码推到 GitHub 仓库。
2. 在仓库设置里开启 GitHub Pages。
3. 选择 `main` 分支或 `docs/` 目录作为发布源。
4. 等待生成访问地址。

### 2. Netlify

适合：

- 想要最省事的静态部署
- 需要后续频繁改页面
- 想直接连 Git 仓库自动发布

优点：

- 配置简单
- 支持自动部署
- 预览和回滚方便

注意：

- 默认域名是平台二级域名
- 国内访问速度不一定稳定

基本步骤：

1. 登录 Netlify。
2. 连接 GitHub 仓库。
3. 选择静态站点发布。
4. 构建命令留空。
5. 发布目录留空或设为项目根目录。

### 3. Vercel

适合：

- 想要快速发布静态页
- 以后可能加一点前端能力
- 想要自动化部署

优点：

- 部署快
- 连接仓库简单
- 预览链接方便分享

注意：

- 默认域名是平台二级域名
- 国内访问体验不一定稳定

基本步骤：

1. 登录 Vercel。
2. 导入 GitHub 仓库。
3. 选择静态站点部署。
4. 不需要写构建命令时就保持默认。
5. 发布完成后拿到访问地址。

### 4. 国内服务器 / 宝塔

适合：

- 主要客户在国内
- 想要访问更稳定
- 计划绑定自己的独立域名

优点：

- 国内访问稳定
- 可绑定自定义域名
- 适合后续扩展成正式官网

注意：

- 通常需要域名
- 中国大陆服务器一般需要备案
- 需要额外服务器和运维成本

基本步骤：

1. 购买服务器和域名。
2. 按接入商要求完成备案。
3. 用 Nginx 或宝塔把 `index.html` 作为站点首页。
4. 把 `assets/` 一起上传到站点目录。
5. 配置 HTTPS。

## 域名与备案建议

### 只想先发出去

- 先用 GitHub Pages、Netlify 或 Vercel。
- 不要先卡在备案上。
- 适合快速验证页面和话术。

### 主要面向国内客户

- 建议尽早准备独立域名。
- 如果使用中国大陆服务器，通常要走备案。
- 如果暂时不想备案，可以先用海外静态托管。

### 适合现在这个项目的顺序

1. 先用 GitHub Pages 或 Netlify 快速上线。
2. 有稳定客户后再换独立域名。
3. 如果后续更重视国内访问，再迁到国内服务器并补备案。

## 上线前检查清单

### 页面检查

- 首页标题、简介、联系方式都正确。
- 微信里能正常打开二维码图片。
- 所有按钮和锚点都能点击。
- 手机端和电脑端都能正常显示。

### 资源检查

- `assets/wechat-qr.jpg` 路径正确。
- 没有引用本地绝对路径。
- 没有依赖构建产物。

### 内容检查

- 联系方式可用。
- 报价和文案没有明显过期信息。
- 没有敏感信息。

### 部署检查

- 发布目录里包含 `index.html`。
- 静态资源一起上传。
- 线上地址可以直接打开。
- 刷新页面不会 404。

### 兼容性检查

- iPhone 浏览器正常。
- Android 浏览器正常。
- Chrome、Safari 访问正常。

## 建议的最短上线流程

1. 本地用 `python3 -m http.server 8080` 检查一次。
2. 选 GitHub Pages 或 Netlify 先发布。
3. 检查线上图片、按钮和二维码。
4. 有正式客户后再考虑独立域名和国内服务器。
